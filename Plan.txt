Этап 0. Подготовка (что нужно до начала)

0.1. Определить формат бота

Реши сразу:
	•	бот только для тебя / друзей / публичный
	•	язык: русский
	•	формат юмора: безопасный, без жести/NSFW

Это важно, чтобы потом не переделывать структуру.

⸻

0.2. Выбрать стек (для старта)

Для простого старта:
	•	Python
	•	библиотека для Telegram-бота (например, aiogram или pyTelegramBotAPI)
	•	хранение данных:
	•	анекдоты — JSON
	•	картинки — папка images/

👉 Для новичка это хороший баланс: не слишком сложно, но уже “по-настоящему”.

⸻

0.3. Создать бота в Telegram

Через BotFather:
	•	создать нового бота
	•	получить токен
	•	сохранить токен в .env (не в коде)

⸻

Этап 1. Проектирование (очень важный этап)

1.1. Прописать функции бота (MVP)

Зафиксируй минимальный набор:

Команды
	•	/start
	•	/help

Кнопки
	•	😂 Анекдот
	•	🖼 Картинка
	•	🎲 Случайное
	•	🔁 Еще

⸻

1.2. Продумать сценарии (как пользователь будет пользоваться)

Сценарий 1: анекдот
	1.	Пользователь нажимает Анекдот
	2.	Бот отправляет случайный анекдот
	3.	Бот показывает кнопку Еще

Сценарий 2: картинка
	1.	Пользователь нажимает Картинка
	2.	Бот отправляет случайную картинку
	3.	Бот показывает кнопку Еще

Сценарий 3: случайное
	1.	Пользователь нажимает Случайное
	2.	Бот выбирает: анекдот или картинка
	3.	Отправляет контент

⸻

1.3. Продумать ограничения (чтобы потом не ловить ошибки)
	•	Что делать, если нет анекдотов?
	•	Что делать, если папка с картинками пустая?
	•	Что делать, если файл картинки битый?
	•	Что делать, если пользователь жмет “Еще” до выбора режима?

Сразу прописать поведение:
	•	“Пока нет анекдотов”
	•	“Пока нет картинок”
	•	“Сначала выбери: анекдот или картинка”

⸻

Этап 2. Подготовка данных (контент)

2.1. Собрать анекдоты

Начни с 20–50 штук.

Рекомендация по формату

Лучше хранить не просто строками, а с запасом на будущее, например:
	•	id
	•	text
	•	category (пока можно "general")
	•	safe (true/false, если захочешь фильтр позже)

Даже если сейчас используешь только текст — такая структура пригодится.

⸻

2.2. Собрать смешные картинки

Создай папку images/ и положи туда 20–30 картинок.

Важно
	•	нормальные форматы: .jpg, .jpeg, .png
	•	без слишком больших файлов (чтобы бот не тормозил)
	•	понятные имена файлов (например meme_01.jpg, meme_02.png)

⸻

2.3. Проверить контент вручную

Перед запуском:
	•	все анекдоты читаются нормально
	•	картинки открываются
	•	нет “битых” файлов
	•	нет спорного контента (если хочешь дружелюбного бота)

⸻

Этап 3. Структура проекта (чтобы не было хаоса)

Сразу разложи проект по папкам. Это очень поможет.

Рекомендуемая структура
	•	bot.py — запуск бота
	•	handlers/ — команды и кнопки
	•	data/
	•	jokes.json
	•	images/ — картинки
	•	utils/ — вспомогательные функции
	•	.env — токен
	•	requirements.txt
	•	README.md

⸻

Этап 4. Реализация v1.0 (по шагам)

Шаг 1. Запуск бота и команда /start

Что должно быть готово:
	•	бот запускается без ошибок
	•	отвечает на /start
	•	показывает приветствие
	•	выводит кнопки меню

Критерий готовности:
Ты открываешь чат → /start → видишь текст и кнопки.

⸻

Шаг 2. Выдача случайного анекдота

Что сделать:
	•	загрузка анекдотов из JSON
	•	случайный выбор одного анекдота
	•	отправка пользователю

Критерий готовности:
Нажимаешь Анекдот → бот присылает шутку.

⸻

Шаг 3. Выдача случайной картинки

Что сделать:
	•	чтение списка файлов из папки images/
	•	случайный выбор файла
	•	отправка картинки пользователю

Критерий готовности:
Нажимаешь Картинка → бот присылает изображение.

⸻

Шаг 4. Кнопка “Случайное”

Что сделать:
	•	бот случайно выбирает тип контента:
	•	анекдот
	•	картинка

Критерий готовности:
Нажимаешь Случайное → каждый раз может прийти разный тип контента.

⸻

Шаг 5. Кнопка “Еще”

Это важная UX-фишка.

Логика

Бот должен помнить, что пользователь смотрел в последний раз:
	•	если был режим “анекдоты” → “Еще” дает еще анекдот
	•	если “картинки” → еще картинку
	•	если “случайное” → снова случайное (или повтор последнего типа — выбери один вариант)

Критерий готовности:
Пользователь может не жать меню заново, а просто жать Еще.

⸻

Шаг 6. Команда /help

Что должно быть:
	•	список команд
	•	кратко что умеет бот
	•	предупреждение про формат контента (например, “только безопасный юмор”)

⸻

Этап 5. Обработка ошибок (обязательно)

Это то, что делает проект “аккуратным”.

5.1. Ошибки контента
	•	jokes.json не найден
	•	jokes.json пустой
	•	папка images/ не существует
	•	в папке нет картинок

5.2. Ошибки пользователя
	•	пользователь нажал кнопку, которую бот не ожидал
	•	пользователь отправил текст вместо команды/кнопки

5.3. Технические ошибки
	•	бот не может отправить файл
	•	ошибка чтения JSON

Что делать
	•	не падать
	•	отправлять понятное сообщение пользователю
	•	писать ошибку в лог (для себя)

⸻

Этап 8. Релиз v1.0

Что считается “готово”

Бот:
	•	запускается без плясок
	•	не падает на базовых сценариях
	•	выдает анекдоты и картинки
	•	имеет меню и помощь

После этого не усложняй сразу — сначала зафиксируй рабочую версию.

⸻

План развития после v1.0 (v1.1 / v2.0)

v1.1 (простые улучшения)
	•	анти-повтор (не показывать только что отправленный контент)
	•	категории анекдотов
	•	подписи к картинкам
	•	команда /random

v1.2
	•	👍 / 👎 оценка
	•	счетчик просмотров
	•	“популярные анекдоты”

v2.0
	•	избранное ⭐
	•	хранение пользователей
	•	SQLite вместо части JSON
	•	админ-панель (простая, через команды)
	•	загрузка новых анекдотов через команду (только для тебя)
